# discrapp — скрэппер реплик персонажей из Discord

Собирает из истории каналов Discord реплики персонажей (embed'ы и/или обычный
текст), **складывает их в базу (SQLite)** и открывает **редактор лога**, где сырую
кашу сообщений доводят до связного повествования: режут склеенные посты, вставляют
и переставляют реплики, размечают сцены и роли, чистят текст пачками — и уже потом
**экспортируют** (Obsidian `.md`, повествование `.md`, `.txt`, `.csv`, `.json`) или
заливают в Nextcloud. Строка текстового результата:

```
[2026-07-19 14:30:05] (Имя персонажа): Текст реплики.
```

## Пайплайн

```
Discord → скрэппинг → БД (прогон + сообщения) → таблица-лог (правки) → экспорт / Nextcloud
```

Каждый скрэппинг — «прогон» (run) в БД; история прогонов сохраняется и доступна
на вкладке **Прогоны**. Столбцы сообщения: чат (ID/имя), реальная дата-время,
автор, текст.

## Три способа использования

| Интерфейс | Файл | Кому |
|---|---|---|
| **Веб-приложение** (React-UI + таблица-лог) | `web.py` + `frontend/` | основной путь |
| Slash-команда `/scrape` в Discord | `bot.py` | быстрый разовый вызов |
| CLI из терминала | `cli.py` | скрипты/автоматизация |

Общее ядро: `core.py` (логика реплик), `discord_rest.py` (Discord REST API),
`nextcloud.py` (WebDAV + публичная ссылка), `config_store.py` (настройки),
`db.py` (SQLite: прогоны, сообщения, журнал отмены), `narrative.py` (текстовые
эвристики: где резать, кто говорит, какая роль), `exporters.py` (сборка документа
из БД). Фронтенд — `frontend/` (React + Vite + TypeScript), собирается в
`frontend/dist` и отдаётся тем же FastAPI.

## Веб-приложение (основной путь)

Возможности UI:
1. список каналов сервера с поиском (включая **ветки и посты форума**);
2. настройка параметров прямо в интерфейсе (ID сервера, токен бота, креды Nextcloud);
3. запуск скрэппинга по выбранным каналам с доп. опциями (даты, режим, предпросмотр);
4. **live-просмотр** собираемых реплик в реальном времени (Server-Sent Events);
5. по завершении — **редактор лога** прогона (см. ниже);
6. **экспорт** в `.md` (Obsidian) / `.md` (повествование) / `.txt` / `.csv` /
   `.json` либо заливка в Nextcloud — уже после правок.

Секреты (токен бота, app-password) хранятся на сервере в `data/config.json`
и **никогда не возвращаются в браузер** — наружу идут только флаги «задан/не задан».

### Редактор лога

Порядок строк задаёт колонка `seq`, а не время: одно сообщение Discord часто
содержит несколько постов, а вставленную вручную реплику ко времени не привязать.
Вставка между соседями берёт среднее их `seq`; когда зазор исчерпан, прогон
перенумеровывается — порядок при этом не меняется.

**Разрезание и сборка** (правая кнопка по тексту сообщения):

- **вырезать выделенное** — сообщение делится на «до / выделенное / после» с
  сохранением порядка; получается 2 или 3 поста с тем же временем. Если кусок
  начинается с «(Имя) - », имя уходит в колонку автора, а префикс убирается;
- **разделить автоматически / по строкам** — для постов вида
  «`**действие**` ⏎ `(Имя) - реплика`» это один клик. «Автоматически» режет по
  строкам, но склеивает обратно соседние строки с одной ролью и одним
  говорящим, чтобы длинная реплика не рассыпалась;
- **объединить** с предыдущим, со следующим или все выделенные — обратная операция;
- **вставить сообщение выше/ниже** (кнопка `＋` на строке), дублировать, переставить.

**Массовые операции** (кнопка 🛠 «Инструменты», `Ctrl+H`) — сначала предпросмотр
«что изменится», потом применение, и всё одним шагом отмены:

| Вкладка | Что делает |
|---|---|
| Разделение | разносит склеенные посты по всему прогону |
| Замена | поиск-замена, подстрокой или регуляркой |
| Чистка | убирает `((OOC))`, упоминания, кастомные эмодзи, markdown-мусор, лишние пробелы |
| Роли | размечает речь / действие / нарратив / OOC по эвристике |

Область действия — выделенные строки либо всё, что попадает под текущие фильтры.

**Структура повествования**: любое сообщение можно объявить началом сцены с
заголовком (в экспорте — `## Заголовок`), а лишнее — **скрыть из экспорта**
вместо удаления, сохранив лог целым. Формат экспорта **«Повествование»** печатает
сцены заголовками, действия курсивом, реплики как «**Имя.** текст», без отметок
времени и без OOC; панель 📖 «Предпросмотр» показывает результат рядом с таблицей.

**Персонажи**: боковая панель с сортировкой **по алфавиту / по количеству реплик /
по первому появлению**, поиском и фильтром по нескольким именам сразу (Ctrl+клик).
Разные написания одного имени сводятся к каноническому («Слить в одного»).

**Отмена**: каждая правка — включая массовые — пишется в журнал; `Ctrl+Z` /
`Ctrl+Shift+Z` или кнопки ↶ ↷. Хранится 100 последних шагов на прогон.

**Клавиши**: `j`/`k` или ↑/↓ — по строкам, `пробел` — выделить, `Esc` — снять
выделение, `Ctrl+F` — поиск, `Ctrl+H` — инструменты, двойной клик — правка ячейки
(`Ctrl+Enter` сохранить, `Esc` отменить).

### Запуск через Docker (рекомендуется)

```bash
cp .env.example .env          # при желании поправить FRAME_ANCESTORS
docker compose up -d --build
```
Открой `http://<хост>:8000`, нажми **⚙ Настройки**, заполни токен бота, ID
сервера и данные Nextcloud → **Сохранить**. Дальше: «Загрузить список» каналов →
выбрать нужные → «Запустить скрэппинг».

> Docker собирает фронтенд внутри образа (multi-stage: `node` → `python`),
> отдельных действий не требуется — `docker compose up -d --build` делает всё.

### Запуск без Docker

Собрать фронтенд один раз (или держать dev-сервер Vite, см. ниже), затем поднять API:

```bash
cd frontend && npm install && npm run build && cd ..   # → frontend/dist
.venv/bin/pip install -r requirements.txt
.venv/bin/uvicorn web:app --host 0.0.0.0 --port 8000
```

FastAPI отдаёт собранный `frontend/dist`; SQLite-база лежит в `DATA_DIR/discrapp.db`.

### Разработка фронтенда

```bash
cd frontend && npm run dev        # Vite на :5173, /api проксируется на :8000
.venv/bin/uvicorn web:app --reload --port 8000   # в соседнем терминале
```

### Встраивание в Nextcloud (вкладкой)

1. Админ ставит приложение **External Sites** в Nextcloud.
2. Добавляет сайт со ссылкой на `http(s)://<хост>:8000`, тип отображения — в iframe.
3. В `docker-compose.yml` (или `.env`) укажи свой origin в `FRAME_ANCESTORS`,
   напр. `'self' https://cloud.example.com` — иначе браузер заблокирует iframe
   (заголовок `Content-Security-Policy: frame-ancestors`).

> Если админ-прав на Nextcloud нет — просто открывай UI по URL, всё работает так же.

## Настройка Discord-бота (один раз, для всех интерфейсов)

Developer Portal → приложение → **Bot** → Reset Token; включи
**Message Content Intent**; добавь бота на сервер (scope `bot`, права
`View Channel` + `Read Message History`).

## Настройка Nextcloud

Настройки → Безопасность → **создать пароль приложения** (app password, не
основной пароль). На стороне Nextcloud ничего ставить не нужно: используются
штатные **WebDAV** (заливка) и **OCS** (публичная ссылка).

## Заметки

- Бот видит только каналы, куда добавлен и где имеет права.
- Один персонажный бот? Укажи его ID в фильтре автора, чтобы не зацепить чужие
  embed'ы (превью ссылок, картинки).
- Голый форум-канал выбрать нельзя — реплики лежат в его постах-ветках, которые
  и выбираются как отдельные каналы.
- Заархивированные ветки Discord может не показывать в списке (ограничение API).
- Веб-приложение желательно держать в доверенной сети/за авторизацией: у него есть
  доступ к токену бота и Nextcloud.
