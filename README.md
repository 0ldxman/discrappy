# discrapp — скрэппер реплик персонажей из Discord → Nextcloud

Собирает из истории каналов Discord сообщения-embed'ы персонажей и складывает
их в текстовый файл на Nextcloud. Персонаж = embed с непустыми `title` (имя) и
`description` (текст). Строка результата:

```
[2026-07-19 14:30:05] (Имя персонажа): Текст реплики.
```

## Три способа использования

| Интерфейс | Файл | Кому |
|---|---|---|
| **Веб-приложение** (UI + встраивание в Nextcloud) | `web.py` | основной путь |
| Slash-команда `/scrape` в Discord | `bot.py` | быстрый разовый вызов |
| CLI из терминала | `cli.py` | скрипты/автоматизация |

Общее ядро: `core.py` (логика реплик), `discord_rest.py` (Discord REST API),
`nextcloud.py` (WebDAV + публичная ссылка), `config_store.py` (настройки).

## Веб-приложение (основной путь)

Возможности UI:
1. список каналов сервера с поиском (включая **ветки и посты форума**);
2. настройка параметров прямо в интерфейсе (ID сервера, токен бота, креды Nextcloud);
3. запуск скрэппинга по выбранным каналам с доп. опциями (даты, фильтры, ссылка);
4. **live-просмотр** собираемых реплик в реальном времени (Server-Sent Events);
5. выбор папки Nextcloud, куда сохранить файл.

Секреты (токен бота, app-password) хранятся на сервере в `data/config.json`
и **никогда не возвращаются в браузер** — наружу идут только флаги «задан/не задан».

### Запуск через Docker (рекомендуется)

```bash
cp .env.example .env          # при желании поправить FRAME_ANCESTORS
docker compose up -d --build
```
Открой `http://<хост>:8000`, нажми **⚙ Настройки**, заполни токен бота, ID
сервера и данные Nextcloud → **Сохранить**. Дальше: «Загрузить список» каналов →
выбрать нужные → «Запустить скрэппинг».

### Запуск без Docker

```bash
.venv/bin/pip install -r requirements.txt
.venv/bin/uvicorn web:app --host 0.0.0.0 --port 8000
```

### Встраивание в Nextcloud (вкладкой)

1. Админ ставит приложение **External Sites** в Nextcloud.
2. Добавляет сайт со ссылкой на `http(s)://<хост>:8000`, тип отображения — в iframe.
3. В `docker-compose.yml` (или `.env`) укажи свой origin в `FRAME_ANCESTORS`,
   напр. `'self' https://cloud.example.com` — иначе браузер заблокирует iframe
   (заголовок `Content-Security-Policy: frame-ancestors`).

> Если админ-прав на Nextcloud нет — просто открывай UI по URL, всё работает так же.

## Настройка Discord-бота (один раз, для всех интерфейсов)

Developer Portal → приложение → **Bot** → Reset Token; включи
**Message Content Intent**; добавь бота на сервер (scope `bot`, права
`View Channel` + `Read Message History`).

## Настройка Nextcloud

Настройки → Безопасность → **создать пароль приложения** (app password, не
основной пароль). На стороне Nextcloud ничего ставить не нужно: используются
штатные **WebDAV** (заливка) и **OCS** (публичная ссылка).

## Slash-команда и CLI (env-конфиг через `.env`)

```bash
.venv/bin/python bot.py                      # затем в Discord: /scrape channel:#канал
.venv/bin/python cli.py 123456789012345678   # --after / --before / --no-upload / --no-share
```

## Заметки

- Бот видит только каналы, куда добавлен и где имеет права.
- Один персонажный бот? Укажи его ID в фильтре автора, чтобы не зацепить чужие
  embed'ы (превью ссылок, картинки).
- Голый форум-канал выбрать нельзя — реплики лежат в его постах-ветках, которые
  и выбираются как отдельные каналы.
- Заархивированные ветки Discord может не показывать в списке (ограничение API).
- Веб-приложение желательно держать в доверенной сети/за авторизацией: у него есть
  доступ к токену бота и Nextcloud.
